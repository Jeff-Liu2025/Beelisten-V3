# 小店模块背包系统优化计划

## 问题分析

1. **背包系统看不见** - `js/shop/index.js` 的 `render` 方法完全替换了容器内容，覆盖了 `index.html` 中预设的 HTML 结构，导致背包按钮丢失事件绑定

2. **道具买了之后没有显示数量** - 商品卡片没有显示已拥有的数量，用户不知道有多少个道具

3. **头像框没有显示** - 装饰类商品购买后没有在个人档案页面显示效果

## 根本原因

- JS 的 `render` 方法使用 `innerHTML` 完全替换了容器，与 HTML 中预设结构不一致
- 背包按钮 `#shopInventoryBtn` 没有绑定点击事件
- 商品卡片渲染时没有显示已拥有数量
- 分类标签切换功能未实现
- 装饰效果未在个人档案页面展示

## 修复方案

### 1. 修改 `js/shop/index.js`

#### 1.1 修改 `render` 方法
- 不再使用 `innerHTML` 完全替换
- 使用 `index.html` 中已有的 DOM 元素
- 正确获取并填充 `#shopGrid`、`#coinAmount`、`#inventoryBadge` 等元素

#### 1.2 添加背包按钮事件绑定
```javascript
bindInventoryButton() {
    const btn = document.getElementById('shopInventoryBtn');
    if (btn) {
        btn.addEventListener('click', () => this.showInventory());
    }
}
```

#### 1.3 修改商品卡片渲染
- 显示已拥有数量（如 `x3`）
- 正确标记已拥有的装饰品

#### 1.4 添加分类标签切换功能
```javascript
bindTabEvents() {
    const tabs = document.querySelectorAll('.shop-tab');
    tabs.forEach(tab => {
        tab.addEventListener('click', (e) => {
            // 切换激活状态，过滤商品
        });
    });
}
```

#### 1.5 修改背包面板显示逻辑
- 使用 `index.html` 中已有的 `#shopInventoryPanel`
- 正确显示道具数量和使用按钮

### 2. 修改 `index.html`

#### 2.1 确保 DOM 结构正确
- 保留现有的商店 HTML 结构
- 确保所有必要的 ID 都存在

### 3. 添加装饰效果展示

#### 3.1 在个人档案页面显示
- 检查用户是否拥有装饰品
- 显示金色边框、徽章等效果

## 文件修改清单

| 文件 | 修改内容 |
|------|----------|
| `js/shop/index.js` | 重构 render 方法，添加事件绑定，显示数量 |
| `index.html` | 确保商店 DOM 结构正确 |
| `css/shop.css` | 添加数量显示样式（如需要） |
