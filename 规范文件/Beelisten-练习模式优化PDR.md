# Beelisten 练习模式优化 — 产品需求文档 (PDR)

> 版本: v1.0 | 日期: 2026-04-22 | 状态: 草案待评审

---

## 1. 项目背景与现状

### 1.1 当前功能
Beelisten v2.1 已完成核心播放功能：
- ✅ 音频播放/暂停/跳转
- ✅ SRT 双语字幕同步高亮
- ✅ 点击字幕跳转对应音频位置
- ✅ 单词即时查字典（划词查询）
- ✅ **词块填空（Chunk Fill）** — 当前唯一练习模式

### 1.2 当前问题
| 问题 | 影响 |
|------|------|
| 训练方式单一 | 用户容易厌倦，学习效果受限 |
| 词块填空偏"考试感" | 机械做题，缺乏真实听力训练 |
| 缺乏听力技巧训练 | 无法训练精听、变速、跟读等核心能力 |
| 没有学习路径 | 用户不知道应该先练什么、再练什么 |
| 无进度追踪 | 不知道自己的弱点和进步 |

### 1.3 目标用户画像
- **身份**：中国大学生（大一~大三），备考四六级或日常提升
- **英语水平**：中等偏下（高考 90~110 分）
- **学习习惯**：每天 30~60 分钟，用手机/电脑自学
- **痛点**：听得懂单词但跟不上整句，想反复听但操作麻烦

## 3. 详细设计方案

### 3.1 ⭐ P0 - AB 循环播放

**核心逻辑**：用户选一段音频范围（A点到B点），循环播放直到手动停止。

**交互设计**：
```
[音频进度条] ▓▓▓▓░░░░░░░░░░░░
             A ↑     B ↑
             [标记A] [标记B] [取消循环]
```

**实现要点**：
- 进度条上点击标记 A/B 点（基于点击位置计算时间）
- `timeupdate` 事件监听：当 `currentTime >= B` 时 `seekTo(A)`
- 视觉反馈：进度条上高亮循环区间
- 快捷键：`[` 标记A，`]` 标记B，`\` 取消循环

**Store 状态**：
```js
Store.set('loopRange', { start: 11.22, end: 15.60, active: true });
```

---

### 3.2 ⭐ P0 - 精听模式（Intensive Listening）

**核心逻辑**：逐句播放 → 用户尝试复述/听写 → 显示原文 → 下一句。

**训练流程**：
```
┌─────────────────────────────────────────┐
│  第 3/18 句                              │
│                                          │
│  [▶️ 播放本句]  ← 点击后播放一次          │
│                                          │
│  [🎤 我能听懂]    [🔄 再听一遍]           │
│                                          │
│  [👁️ 显示原文]  ← 点击后展开             │
│     "I never really thought about        │
│      what I wanted to do with my life."  │
│                                          │
│  [✅ 标记已掌握] → 进入下一句              │
└─────────────────────────────────────────┘
```

**两种子模式**：
1. **理解模式**：听完选择"听懂了 / 没听懂"，没听懂可再听3次后显示原文
2. **听写模式**：听完在输入框中打出听到的句子，系统用 Levenshtein 距离比对

**实现要点**：
- 复用 subtitle 模块的句子分割能力
- 按 `startTime` 播放单句后自动暂停（监听 `timeupdate`，到达 `endTime` 时 pause）
- 利用现有的 word-selected 事件机制，精听时禁用字幕高亮
- 听写比对算法：
  ```js
  function similarity(str1, str2) {
    // 忽略大小写、标点，计算编辑距离
    const s1 = str1.toLowerCase().replace(/[^a-z]/g, '');
    const s2 = str2.toLowerCase().replace(/[^a-z]/g, '');
    return levenshtein(s1, s2);
  }
  ```

### 3.3 ⭐ P0 - 变速播放

**核心逻辑**：用户可以调节音频播放速度（0.5x ~ 2.0x），系统记住偏好。

**交互设计**：
```
速度: [0.5x] [0.75x] [1.0x] [1.25x] [1.5x] [2.0x]
           ↑ 默认
```

**实现要点**：
- HTML5 Audio 原生支持：`audio.playbackRate = 0.75`
- 注意：变速后音调会变（Chipmunk 效果）
- 可选：使用 Web Audio API 的 `AudioContext` + `playbackRate` + `pitchShift` 保持音调（Phase 2）
- 默认推荐：难句子用 0.75x，复习用 1.25x

**Store 状态**：
```js
Store.set('playbackRate', 1.0);
```

---

### 3.4 P1 - 全文听写（Dictation）

**核心逻辑**：完整听写整段音频，逐句输入，系统评分。

**与精听模式的区别**：
| | 精听模式 | 全文听写 |
|:-|:---------|:---------|
| 粒度 | 单句 | 全文 |
| 反馈 | 即时显示原文 | 提交后整体评分 |
| 难度 | 中 | 高 |
| 适合阶段 | 中级 | 中高级 |

**实现要点**：
- 界面：左侧音频播放器，右侧文本输入区
- 每句一个 `<textarea>`，按时间顺序排列
- 提交后系统逐句比对，标红错误单词
- 显示分数：`正确单词数 / 总单词数`

---

### 3.5 P1 - 影子跟读（Shadowing）

**核心逻辑**：音频播放的同时，用户跟读，录音后对比自己的发音和原文。

**技术难点**：
- 需要 Web Speech API（`SpeechRecognition`）或 Web Audio API 录音
- 浏览器兼容性：Chrome 支持较好，Firefox 部分支持
- 发音评分需要 AI 模型（本地实现较困难）

**简化版方案（推荐先实现）**：
- 只提供"跟读练习"界面：音频播放时显示波形/字幕，用户跟着读
- 不提供自动评分，改为"自评"（流畅/卡顿/不会）
- 记录跟读过的句子，方便复习

---

### 3.6 P1 - 学习报告面板

**核心逻辑**：记录用户的练习数据，生成可视化报告。

**记录数据**：
```js
// localStorage 存储结构
{
  "beelisten-stats": {
    "daily": {
      "2026-04-22": {
        "totalMinutes": 35,
        "sessions": 2,
        "wordFillAccuracy": 0.72,
        "intensiveCompleted": 12,  // 精听完成句数
        "wordsLookedUp": ["ambitious", "career"]
      }
    },
    "weakWords": ["thought", "through", "though"],  // 经常混淆的词
    "totalListenedMinutes": 420
  }
}
```

**报告内容**：
- 本周学习时长折线图
- 各模式使用占比饼图
- 高频错题词云
- 连续学习天数（ streak ）

---

### 3.7 P2 - 生词本

**核心逻辑**：自动收集用户查过的单词，生成个人词库。

**功能**：
- 查词时自动加入生词本（可开关）
- 生词本页面：单词 / 音标 / 释义 / 例句 / 来源音频
- 支持标记"已掌握"
- 导出为 Anki 格式（.apkg）

## 4. 学习路径设计

**建议的默认学习流程**（用户选择一篇文章后）：

```
Step 1: 词汇预习（2分钟）
   └─> 浏览生词，听发音
   
Step 2: 泛听理解（5分钟）
   └─> 1.0x 速度完整听一遍，选择整体理解程度
   
Step 3: 精听训练（15分钟）
   └─> 逐句精听（0.75x），直到听懂每句
   
Step 4: 词块填空（10分钟）
   └─> 巩固词汇和搭配
   
Step 5: 影子跟读（10分钟）
   └─> 模仿语音语调
   
Step 6: 全文听写（可选，高阶）
   └─> 终极检验
```

**用户可自由跳过步骤**，但系统推荐按此顺序。

## 6. 词块填空模式优化建议

当前词块填空可以保留，但建议以下改进：

1. **增加语境提示**：填空时显示前后句，避免孤立猜词
2. **答案模式分级**：
   - 简单：显示首字母 + 词性
   - 中等：只显示词性
   - 困难：无任何提示
3. **错题复习**：自动收集填错的词块，生成"错题重练"
4. **听音填空**：先播放音频（不显示字幕），让用户听音填空

## 9. 风险评估

| 风险 | 等级 | 缓解方案 |
|------|:----:|:---------|
| Web Speech API 兼容性问题 | 中 | 跟读功能标记为实验性功能，提供降级方案 |
| 纯前端数据持久化限制 | 低 | 使用 localStorage + IndexedDB，定期导出备份 |
| 代码复杂度上升 | 中 | 严格模块化，每个训练模式独立文件，统一接口 |
| 用户学习负担增加 | 低 | 默认推荐流程，用户可自由跳过步骤 |
